# 📚 Mission Control - 用户指南

## 🎯 产品概述

Mission Control 是一个智能需求分析和技术文档生成平台，通过AI技术帮助团队快速分析需求、生成技术文档、做出数据驱动的技术决策。

### 核心功能
1. **智能需求分析**: 上传需求文档，自动提取功能点和技术要求
2. **AI增强分析**: 使用Grok AI进行深度语义分析和智能推荐
3. **文档自动生成**: 一键生成SRS、TDD、项目计划等标准技术文档
4. **可视化决策**: 交互式仪表板展示分析结果和推荐方案
5. **性能监控**: 实时系统性能监控和优化建议

## 🚀 快速开始

### 1. 访问系统
- **生产环境**: http://localhost:3001
- **开发环境**: http://localhost:3000

### 2. 首次使用
1. **上传需求文档**: 支持.txt, .md, .html, .docx, .pdf格式
2. **或输入文本**: 直接在文本框中输入需求内容
3. **开始分析**: 点击"开始分析"按钮
4. **查看结果**: 在可视化仪表板中查看分析结果
5. **生成文档**: 一键生成完整的技术文档

### 3. 界面导航
- **首页**: 文档上传和分析入口
- **分析结果**: 可视化展示分析结果
- **文档库**: 查看和下载生成的文档
- **监控面板**: 系统性能监控
- **设置**: 系统配置和个性化设置

## 🔧 系统配置

### 环境要求
- **操作系统**: Linux, macOS, Windows
- **Node.js**: 18.x 或更高版本
- **内存**: 至少 2GB RAM
- **存储**: 至少 500MB 可用空间
- **网络**: 稳定的互联网连接

### 安装方式
#### 方式一: Docker部署 (推荐)
```bash
# 克隆代码
git clone <repository-url>
cd mission-control

# 配置环境变量
cp .env.example .env.production
# 编辑 .env.production 文件

# 启动服务
./scripts/deploy-production.sh
```

#### 方式二: 手动部署
```bash
# 安装依赖
npm ci

# 构建应用
npm run build

# 启动服务
npm run start
```

### 配置选项
#### 环境变量
```bash
# 必需配置
NODE_ENV=production
PORT=3001

# AI服务配置
GROK_API_KEY=your_grok_api_key
OPENAI_API_KEY=your_openai_api_key

# 安全配置
JWT_SECRET=your_jwt_secret
SESSION_SECRET=your_session_secret

# 数据库配置
DATABASE_URL=file:./data/production.db
```

#### 功能开关
```bash
# 启用/禁用功能
ENABLE_AI_ANALYSIS=true
ENABLE_DOCUMENT_GENERATION=true
ENABLE_VISUALIZATION=true
ENABLE_PERFORMANCE_MONITORING=true
```

### 备份和恢复
#### 数据备份
```bash
# 备份数据库
cp ./data/production.db ./backups/production-$(date +%Y%m%d).db

# 备份上传文件
tar -czf ./backups/uploads-$(date +%Y%m%d).tar.gz ./uploads/
```

#### 系统恢复
```bash
# 恢复数据库
cp ./backups/production-20240223.db ./data/production.db

# 恢复上传文件
tar -xzf ./backups/uploads-20240223.tar.gz -C ./
```
