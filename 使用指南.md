# Mission Control 智能需求分析系统使用指南

## 🚀 系统访问地址

### 1. Mission Control 主面板
- **地址**: http://localhost:3002
- **功能**: 系统总览、快速操作、数据监控、工具管理

### 2. 智能需求分析系统
- **地址**: http://localhost:3002/requirements-analysis
- **功能**: 需求分析、技术栈推荐、风险评估、文档生成

### 3. API 服务器
- **地址**: http://localhost:3001
- **功能**: 后端API服务、数据存储、业务逻辑处理

## 📋 智能需求分析系统功能

### 核心功能
1. **Markdown格式支持** - 支持完整的Markdown语法编辑和预览
2. **文件上传** - 支持TXT、MD、HTML、DOCX、PDF格式
3. **智能分析** - 自动分析需求、提取功能点
4. **技术栈推荐** - 基于需求推荐合适的技术栈
5. **风险评估** - 识别项目风险并提供缓解方案
6. **工作量估算** - 自动估算开发时间和团队规模
7. **文档生成** - 一键生成SRS、TDD、项目计划等文档

### 使用步骤
1. **访问需求分析页面**: 点击Mission Control主面板的"需求分析"按钮
2. **输入需求**: 使用Markdown格式编写需求文档
3. **分析需求**: 点击"分析需求"按钮
4. **查看结果**: 查看分析结果、技术推荐、风险评估
5. **生成文档**: 点击"生成技术文档"获取完整文档

## 🔧 API 接口说明

### 健康检查
```bash
GET /api/health
```
返回系统健康状态

### 需求分析服务状态
```bash
GET /api/requirements-analysis?action=status
```
返回服务能力和支持格式

### 需求分析
```bash
POST /api/requirements-analysis
```
参数:
- `text`: 需求文本内容（Markdown格式）
- `file`: 上传的需求文档文件
- `generateDocs`: 是否生成文档（true/false）

### 监控指标
```bash
GET /api/metrics
```
返回Prometheus格式的系统监控指标

## 🚨 常见问题解决

### 1. 页面无法访问
- 检查端口3002是否被占用
- 重启开发服务器：`npm run dev`
- 检查日志：`tail -f /tmp/mission-control-dev.log`

### 2. API调用失败
- 检查API服务器是否运行：`curl http://localhost:3001/api/health`
- 检查端口3001是否被占用
- 重启API服务器（如果独立运行）

### 3. 上传功能无效
- 检查文件格式是否支持（TXT、MD、HTML、DOCX、PDF）
- 检查文件大小（最大100MB）
- 检查浏览器控制台是否有错误

### 4. Markdown预览不显示
- 检查是否安装了`marked`和`highlight.js`依赖
- 重启开发服务器
- 清除浏览器缓存
