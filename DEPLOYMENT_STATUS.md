## 🚀 部署选项

### 选项1: Docker部署 (推荐)
```bash
# 一键部署
./deploy.sh docker

# 手动部署
docker-compose up -d
```

**优点**:
- 环境一致性
- 快速部署
- 易于扩展
- 完整的服务栈

### 选项2: 传统部署
```bash
# 生产部署
./deploy.sh production

# PM2进程管理
pm2 start npm --name "mission-control" -- start
```

**优点**:
- 直接控制
- 资源占用少
- 熟悉的工作流

### 选项3: Vercel部署
```bash
# Vercel部署
./deploy.sh vercel

# 或手动
vercel --prod
```

**优点**:
- 完全托管
- 自动SSL
- 全球CDN
- 简单易用

## 🔧 故障排除指南

### 常见问题

#### 1. 服务器无法启动
```bash
# 检查错误
npm run dev 2>&1 | grep -A5 "error"

# 清理缓存
rm -rf .next node_modules
npm install
```

#### 2. 数据库连接失败
```bash
# 测试连接
node -e "测试数据库连接代码"
```

#### 3. 端口冲突
```bash
# 检查端口占用
lsof -i :3000

# 停止占用进程
kill -9 <PID>
```

#### 4. 内存不足
```bash
# 监控内存
docker stats

# 或使用Node.js内存限制
node --max-old-space-size=4096 server.js
```
